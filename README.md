# Go-Example
practice code for Go language

提交类型参考：
- feat：新功能（feature）添加或实现。
- fix：修复 bug。
- refactor：代码重构，既不修复 bug 也不添加新功能。  
- docs：文档相关的变更，如文档添加、更新或删除。  
- style：代码样式和格式调整，不影响代码逻辑。  
- test：添加或修改测试代码。  
- chore：与构建过程或辅助工具相关的变更。  
- perf：性能优化相关的代码更改。  
- revert：撤销之前的提交。  
- config：对配置文件的更改。  
- merge：分支合并。
